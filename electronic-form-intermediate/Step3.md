# Step3：HTML生成

以下のプロンプトは、Step2 で作成した「フォーム視覚ドラフト」を  
**HTML5 ＋ ローカルCSSのみ** で構成された静的HTMLに変換するためのものです。

必ず、直前の Step2 で生成した **フォーム視覚ドラフト** を元にしてください。
分割して出力された場合は、Step2の各分割出力内の フォーム視覚ドラフト を正確に抽出し、ひとつの「視覚ドラフト全文」として元にしてください。
同時に参照用として **「フォームUI選択辞書」** も添付されている前提とします。

---

## 【目的】

- Step2 のフォーム視覚ドラフトを、**構造を変えずに** HTML5 に変換する  
- CSSのみで装飾した、スマートフォンでも読みやすいフォームレイアウトを作る  
- class名やstyleの指定は、「フォームUI選択辞書 ＞ 説明文・装飾パーツ（Markdown/HTML対応）」の方針に合わせる  
- 外部サイトに依存しない、**1ファイル完結のローカルHTML** を生成する  

---

## 【生成するHTMLの条件】

- 原則として、`<!DOCTYPE html>` から `</html>` までを含む **HTML5文書** として出力してください  
  - ただし、ツールや運用上の制約がある場合は、`<form>〜</form>` 部分のみの出力でも構いません
- `<meta charset="UTF-8">` を含めること
- CSSは `<head>` 内の `<style>...</style>` に記述し、**外部ファイルやCDNは使用しないこと**
- JavaScript（`<script>`）は使用しないこと  
  - 分岐ロジックや選択上限などは、HTMLのレイアウト・テキスト・コメントで静的に表現する
- class名や基本的なスタイルは、「フォームUI選択辞書 ＞ 説明文・装飾パーツ（Markdown/HTML対応）」に沿って決めること  
  - 例：`.page-title`, `.section-title`, `.field-label`, `.field-input`, `.help-text`, `.note` など  
  - まったく新しい思想のclass名を自由に発明せず、辞書の方向性に合わせる

---

## 【記号 → HTMLタグのマッピング】

Step2 の視覚ドラフトに含まれる記号を、以下のルールでHTMLタグに変換してください。

| 視覚ドラフトの記号 | HTMLタグ例（構造イメージ） |
|--------------------|----------------------------|
| `[＿＿＿＿＿＿]` | `<input type="text" class="field-input">` |
| `[123]` | `<input type="number" class="field-input">` |
| `⦿ 選択肢A　○ 選択肢B` | `<input type="radio">` を選択肢ごとに並べる |
| `☑ 選択肢A　□ 選択肢B` | `<input type="checkbox">` を選択肢ごとに並べる |
| `[選択してください ▼]` | `<select class="field-input">…</select>` |
| `[📅 日付を選択]` | `<input type="date" class="field-input">` |
| `[⏰ 時間を選択]` | `<input type="time" class="field-input">` |
| `[画像アップロード]` | `<input type="file" class="field-input">` |
| `[📍地図入力]` | 住所用 `<input type="text">` ＋ 地図入力に関する補足テキスト |
| `[✍ 手書きエリア]` | `<textarea class="field-input handwrite-area"></textarea>` など |
| Markdown形式の表（グリッド） | `<table>`, `<thead>`, `<tbody>` でテーブルとして変換 |

※ `name` / `id` 属性は付与してよいが、主目的は「見た目のプレビュー」であり送信機能は不要です。

---

## 【分岐・選択上限・ラジオ縦横・グリッドの表現】

- ラジオボタンやチェックボックスの **縦並び／横並び** は、  
  適切なクラスやCSSでレイアウトを調整しつつ、**静的なHTML** として表現してください
- 「1行につき1つまで」「最大3つまで選択可」などの **選択上限や制約** は、  
  JavaScriptではなく、以下のいずれかで表現します：
  - ラベルや補足テキスト（例：`<p class="note">（最大3つまで選択できます）</p>`）  
  - HTMLコメント（例：`<!-- 1行につき1つまで選択 -->`）
- 振込方法など、**選択肢によって後続の入力項目が変わる分岐ロジック** は、  
  - 分岐ごとに対応する入力欄をすべてHTML上に記述した上で  
  - 「〇〇を選んだ場合のみ入力」などの説明文やコメントで条件を明記してください  
  例：  
  `<!-- ゆうちょ選択時のみ使用 -->` や  
  `<p class="help-text">※「銀行振込」を選んだ場合のみ入力してください。</p>`
- 項目によってはレイアウトの改善で、項目を2カラム、3カラムに並べる判断をしても良い
  - 例）[氏][名]、[氏ふりがな][名ふりがな]、[郵便番号][都道府県]
  - 注意点：
  - 1項目が長文説明の場合は自動で縦積みにしてもよい
  -レスポンシブは不要（固定幅でプレビュー用でOK）

---

## 【バリデーションの表現ルール】

- バリデーションは **HTML標準属性の範囲** で表現してください  
  - 例：`type`（text / number / email / date 等）、`required`、`min` / `max`、`maxlength`、`pattern` など  
- JavaScriptによるバリデーション処理は行いません  
- Step2 の視覚ドラフトで「必須」とされている項目は、`required` を付与して構いません

---

## 【テキスト（説明文・注意文）の扱い】

- Step2 に書かれている以下のテキストは、すべてHTML上に反映してください：
  - ページ冒頭の説明文  
  - 各セクションの説明文  
  - 項目ごとの補足・注意書き（例：「不明な場合は『その他』を選択してください」）  
- 説明文の種類に応じて、フォームUI選択辞書にあるクラス・タグを使い分けてください  
  - 一般的な説明 → `class="help-text"` など  
  - 注意喚起 → `class="note"` やそれに準じるクラス  
- 太字・強調が必要な場合も、辞書の方針に合わせて `<strong>` や `<span class="…">` を使用してください

---

## 【出力ルール】

1. **1つのコードブロック内に、HTMLをまとめて出力してください。**  
   - 途中で途切れそうな場合のみ、「前半／後半」などに分割しても構いません  
2. フォームが長く、一度にすべて出力できない場合は：
   - AI側の判断で「前半」「後半」などに自動分割してよい  
   - 各ブロック末尾に `（続きます）` と記載し、最後のブロックの末尾に `以上でStep3完了です` と記載してください
3. Step2 の出力のうち、**HTML化の対象は「フォーム視覚ドラフト」の部分のみ** とします  
   - 「改善視点」「UIパーツ選定理由」などの解説部分はHTMLにしないでください
4. Step2 に含まれる項目・説明文・表は、**1つも省略せず** HTML に反映してください  

---

## 【禁止事項】

- Bootstrap などの外部CSS/JS/CDNの利用  
- `<link>` による外部CSSの読み込み  
- `<script>` による動的処理やバリデーション  
- Step2 に存在しない項目や説明文を勝手に追加すること  
- 項目の削除や結合など、フォームの構造・順序を勝手に変更すること
