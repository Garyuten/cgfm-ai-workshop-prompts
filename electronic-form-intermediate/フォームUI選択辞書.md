# 📘フォームUI選択辞書

※生成AIに対して「適切なUIパーツで改善提案を行わせる」ための参照辞書
※ブラウザ標準のフォームパーツに加え、一般的な自治体向け電子申請システムなら利用可能なUIパーツを追加

---

## ✅ UIを選ぶための基本判断軸

| 判断観点 | YESなら | NOまたは別条件なら |
| --- | --- | --- |
| 入力が短い？ | 単一行テキスト | 複数行テキスト |
| 自由入力より選択が適切？ | ラジオ／チェック／プルダウン | テキスト入力 |
| 選択肢が1つだけ？ | ラジオ | チェック or グリッド |
| 選択肢が多い（6件以上）？ | プルダウン | ラジオ |
| 行ごとに比較・選択必要？ | グリッド型 | 個別ラジオ／チェック |
| 繰り返し行がある？ | 表形式入力 | 単項目入力 |
| 郵便・位置情報が必要？ | 住所・地図・GPS系 | 通常入力 |
| 法的確認や署名が必要？ | チェック・手書き署名 | 任意入力 |

---

## ✍ 入力系パーツ

| パーツ | 使うとき | 避けるとき |
| --- | --- | --- |
| 単一行テキスト | 氏名・電話など短文 | 長文説明 |
| 複数行テキスト | 意見・自由記述 | 定型回答 |
| 数値入力 | 金額・人数 | 半角/全角混同による誤認を避けたい場合（セレクト式検討） |
| 特定形式入力（住所・郵便番号等） | 入力ミスを防ぐ | 任意表現を求める場合 |
| 手書き入力（Canvas） | 署名・図解が必要 | 長文記述 |

---

## ✅ 選択系（単一／複数）

| パーツ | 特徴 | 適切なケース |
| --- | --- | --- |
| ラジオ（縦・横） | 1つだけ選択 | 選択肢3～5件程度 |
| ラジオ（ランダム） | バイアス回避 | アンケート系 |
| プルダウン | 多数候補 | 都道府県選択など |
| チェックボックス | 複数選択可 | 制限なしの場合 |
| チェック＋上限設定 | 「最大3つまで」など制御 | 複数選択に制限がある場合 |

---

## 📊 グリッド・表形式（比較・複合入力）

| パーツ | 使いどころ |
| --- | --- |
| グリッド（行ごと1つ選択） | 「選択項目 × 評価項目」のように1行1択 |
| グリッド（行ごと複数選択） | 「該当する項目が複数ある場合」 |
| 表形式入力（行固定） | 項目を表形式で並べたいとき |
| 表形式入力（行追加可） | 明細や複数人分を登録させるとき |

---

## 📢 説明文・装飾パーツ（Markdown/HTML対応）

✅ リッチテキスト入力では以下利用可：

| 表現 | 記法例 |
| --- | --- |
| 見出し | `# ～ ######` |
| 太字 | `**太字**` / `<strong>` |
| 箇条書き | `-` / `1.` |
| 引用 | `>` |
| 表 | マークダウン記法の表 |
| 色／サイズ変更 | `<p style="color:red;">` |
| 詳細折りたたみ | `<details><summary>...</summary>...</details>` |
| 画像 | `<img src="URL" alt="" width="△△">` |

📍画像は事前アップロード済URLを使用可能。

## 📌 改善提案時に意識させたいポイント（AI向け指示に使う）

✅ 入力負荷は適切か  
✅ ミスを防げるUIか  
✅ 比較しやすいか  
✅ 長文説明は適切に構造化されているか（見出し・箇条書き・画像化）  
✅ 説明と入力UIの対応が視覚的にわかりやすいか  
✅ 行政文書として信頼性が保たれるか（署名・同意・根拠表示）
